## 一、需求說明書（SRS）

### 1.1 系統目的
本系統旨在建置一套完整且具可擴展性的企業人事管理系統，協助企業對員工資料、人事流程與行政管理進行系統化管理。透過資訊系統取代傳統紙本或人工操作，可有效降低人為錯誤、提升資料一致性，並縮短人事作業流程，使企業能更專注於核心業務發展。

### 1.2 使用者角色
- **人資管理員**：負責員工資料管理、人事異動及薪資管理。  
- **一般員工**：可查詢及維護個人基本資料，查看出勤與薪資資訊。  
- **系統管理員**：負責帳號與權限管理，進行系統維護。  

### 1.3 功能需求
- 員工基本資料管理  
- 人事異動管理  
- 出勤紀錄管理  
- 薪資資料管理  
- 使用者登入與權限控管  

### 1.4 非功能需求
- **效能**：一般操作回應時間應小於 3 秒  
- **安全性**：具帳號驗證與權限控管機制  
- **可用性**：介面直覺易用，降低使用者學習成本  
- **可維護性**：模組化設計，方便未來擴充與維護

## 二、概要設計說明書（HLD）

### 2.1 系統架構
本系統採三層式架構設計：
- **表現層（Presentation Layer）**：提供使用者操作介面  
- **應用層（Application Layer）**：處理系統核心業務邏輯  
- **資料層（Data Layer）**：負責資料儲存與存取，確保資料完整性  

### 2.2 模組劃分
- 使用者管理模組  
- 員工資料管理模組  
- 出勤管理模組  
- 薪資管理模組

### 2.4 設計原則與假設條件
本系統在設計階段遵循以下軟體工程設計原則，以確保系統具備良好的結構性與可維護性：

1. **模組化設計原則**  
   將系統功能依照業務性質切分為不同模組，如使用者管理、員工資料、出勤與薪資模組，降低模組間耦合度，提升系統可維護性。

2. **角色與權限分離原則**  
   系統依使用者角色（人資管理員、一般員工、系統管理員）進行權限控管，確保不同角色僅能操作授權範圍內之功能，以提升系統安全性。

3. **資料一致性與完整性原則**  
   所有核心資料皆透過系統流程進行存取，避免直接操作資料庫，以確保資料正確性與一致性。

本系統設計時亦基於以下假設條件：
- 系統使用者皆具備基本電腦操作能力  
- 系統主要於企業內部網路或受信任環境中使用  
- 使用者人數屬於中小型企業規模，系統負載相對穩定

## 三、詳細設計說明書（LLD）

### 3.1 模組功能說明
- **使用者管理模組**：負責登入驗證及權限控管  
- **員工資料模組**：提供員工資料新增、修改及查詢功能  
- **出勤管理模組**：記錄員工上下班時間，提供查詢與統計功能  
- **薪資管理模組**：管理員工薪資資料，提供查詢功能  

### 3.2 核心流程示例
以新增員工資料流程為例：
1. 人資管理員於系統介面輸入員工資料  
2. 系統檢查資料完整性與格式  
3. 驗證通過後，資料寫入資料庫  
4. 系統回傳新增成功或失敗訊息

### 3.4 系統擴充與未來發展方向
本系統於設計階段即保留未來擴充彈性，後續可依企業需求逐步擴充相關功能，包含但不限於：

- **請假與排班管理模組**：整合出勤管理，提供請假申請與班表設定功能。
- **績效考核模組**：結合員工資料與出勤資訊，作為績效評估依據。
- **報表與統計分析功能**：提供人資管理決策所需之統計報表。
- **多平台支援**：未來可擴充為行動裝置或雲端系統，提高系統可用性。

透過模組化與分層式架構設計，上述功能皆可在不影響既有系統運作的前提下進行擴充，符合軟體工程對系統演進性的要求。
