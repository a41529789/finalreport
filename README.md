## 一、需求說明書（SRS）

### 1.1 系統目的
本系統旨在建置一套完整且具可擴充性的企業人事管理系統，協助企業有效管理員工資料、人事流程與行政作業。透過資訊系統取代傳統紙本或人工處理方式，不僅可降低人為操作錯誤，亦能提升資料一致性與處理效率，使人資管理流程更加系統化，進而協助企業專注於核心業務發展。

### 1.2 使用者角色
- **人資管理員**：負責員工資料維護、人事異動管理及薪資相關作業。  
- **一般員工**：可查詢與維護個人基本資料，並查看出勤與薪資資訊。  
- **系統管理員**：負責帳號管理、角色權限設定與系統維護工作。  

### 1.3 功能需求
- 員工基本資料管理  
- 人事異動管理  
- 出勤紀錄管理  
- 薪資資料管理  
- 使用者登入與權限控管  

### 1.4 非功能需求
- **效能需求**：一般操作回應時間應小於 3 秒  
- **安全性需求**：具備帳號驗證與角色權限控管機制  
- **可用性需求**：介面設計直覺易用，降低使用者學習成本  
- **可維護性需求**：採模組化設計，利於後續系統擴充與維護

## 二、概要設計說明書（HLD）

### 2.1 系統架構
本系統採用三層式架構（Three-Tier Architecture）設計，將系統職責明確分層，以提升系統可維護性與擴充性：
- **表現層（Presentation Layer）**：負責使用者操作介面與資料顯示  
- **應用層（Application Layer）**：處理系統核心業務邏輯與流程控制  
- **資料層（Data Layer）**：負責資料儲存、存取與資料完整性維護  

### 2.2 模組劃分
- 使用者管理模組  
- 員工資料管理模組  
- 出勤管理模組  
- 薪資管理模組

### 2.4 設計原則與假設條件
為確保系統具備良好品質，本系統於設計階段遵循以下原則：

1. **模組化設計原則**  
   將系統依功能性質劃分為多個模組，降低模組間耦合度，提升系統可維護性。

2. **角色與權限分離原則**  
   依使用者角色進行權限控管，確保各角色僅能操作其授權範圍內的功能，以提升系統安全性。

3. **資料一致性與完整性原則**  
   所有資料存取皆透過系統流程進行，避免直接操作資料庫，以確保資料正確性。

本系統設計時亦基於以下假設條件：
- 使用者具備基本電腦操作能力  
- 系統主要於企業內部或受信任環境中使用  
- 使用人數規模以中小型企業為主，系統負載相對穩定

## 三、詳細設計說明書（LLD）

### 3.1 模組功能說明
- **使用者管理模組**：提供登入驗證、角色識別與權限控管功能  
- **員工資料模組**：負責員工資料之新增、修改、刪除與查詢  
- **出勤管理模組**：記錄員工上下班時間，並提供查詢與統計功能  
- **薪資管理模組**：管理員工薪資資料，供人資管理與查詢使用  

### 3.2 核心流程示例
以「新增員工資料」流程為例：
1. 人資管理員於系統介面輸入員工基本資料  
2. 系統進行資料格式與完整性檢查  
3. 驗證通過後，將資料寫入資料庫  
4. 系統回傳新增成功或失敗之結果訊息

本系統於設計階段即預留擴充彈性，未來可依企業需求進行功能延伸，包括但不限於：
- **請假與排班管理模組**  
- **績效考核與評估模組**  
- **報表與統計分析功能**  
- **行動裝置或雲端平台支援**
